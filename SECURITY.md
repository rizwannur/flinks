# Security Policy

## Supported versions

This project follows semver. Security fixes land on the latest published
minor. Older lines are not backported unless a fix is trivial.

## Reporting a vulnerability

**Do not open a public issue for security problems.**

Report privately via GitHub's [private vulnerability reporting](https://github.com/rizwannur/flinks/security/advisories/new),
or email the maintainer (see the `author` field in `package.json`).

Please include:

- A description of the issue and its impact
- Steps to reproduce (a minimal snippet is ideal)
- Affected version(s)

You'll get an acknowledgement within 72 hours. Once a fix is ready, we'll
coordinate a release and credit you in the advisory unless you prefer to stay
anonymous.

## Scope

This is an **unofficial** client for the Flinks API. Vulnerabilities in
Flinks' own service should be reported to [Flinks](https://flinks.com)
directly — this policy covers only the code in this repository.
